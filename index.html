NetInfo

The network inspector for Safari. See exactly what's happening behind every website you visit.

View on the Mac App Store

Everything you need to know

One click reveals a full breakdown of the current page's connection.

🔍

IP Address & Family

See the resolved server IP address and whether the connection is IPv4 or IPv6. Copy the IP to clipboard with one tap.

🔒

TLS & Certificate

View the TLS version in use (1.0–1.3) and the SSL certificate expiry date — colour-coded so you spot expiring certs instantly.

🛡

HSTS Detection

Instantly see whether a site enforces HTTP Strict Transport Security to prevent downgrade attacks.

⚡️

HTTP Protocol

See whether the site is served over HTTP/1.1, HTTP/2, or the latest HTTP/3 protocol.

📡

ASN & ISP

Identify the Autonomous System Number and internet service provider hosting the site.

⏱

Performance Timings

DNS lookup, TCP connect, TLS handshake, and time to first byte — all measured in milliseconds.

Your privacy, guaranteed

NetInfo collects no personal data, stores nothing, and never tracks your browsing. All connection data stays on your device. ISP lookups use the public ipapi.co API with your IP address only.

Support

How do I enable NetInfo?

Launch the NetInfo app, then open Safari → Settings → Extensions and enable NetInfo. You may also need to go to Safari → Develop → Allow Unsigned Extensions if using a development build.

The toolbar icon is not visible

Right-click the Safari toolbar → Customise Toolbar, then drag the NetInfo icon into the toolbar.

Why do some timing values show —?

DNS, TCP and TLS timings are only available on fresh connections. If Safari reuses an existing connection to the server, these values will not be available. Try refreshing the page.

Does NetInfo work on all websites?

NetInfo works on all HTTPS and HTTP websites. Certificate and TLS details are only available on HTTPS sites.

What macOS version is required?

NetInfo requires macOS 10.15 Catalina or later.
